Specific Folder Restore

When you restore a file to a specifc folder location, you have the option of recreating the parent folders' structure or restoring the file, individually, to that location. If you choose to recreate the parent folders, the file will be restored with its original folder hierarchy, meaning the parent folders of that file will be restored as well. Only the folder structure gets restored, not any files within the folder structure - except, of course, the file you chose to be restored. An example of each is shown below.

Restoring a file and recreating its parent folders:

  1. In this example, the "addSchedule.jpg" file will be restored. From the graphic below, you can see that this file, which is highlighted, is in the "images" folder. This folder is within the "ChronoSync_Help_v4.0" folder. So, the path of "addSchedule.jpg" is: ChronoSync_Help_v4.0/images.
  1. To restore this file with the entire path (ChronoSync_Help_v4.0/images) intact, choose "Restore->From Archive" from the "Actions" menu, select "Specific Folder" on the "Restore from Archive" sheet and choose "Restore".
  2. You should now see the file browse sheet. Navigate to the location you wish to restore the selected file. In this example, "addSchedule.jpg" will be restored to the Desktop. Make sure "Recreate parent folders" is checked. By default, it is.
  1. Once you choose "Select", a restore verification sheet will be displayed.
  1. The Finder window graphic below shows the file and its parent folders after restoring it. As you can see, the folder structure was retained and restored along with the selected file. Note that no other files that may have been within these folders were restored. Just the one you had selected.

Restoring a file individually (not recreating its parent folders):

  1. In this example, the "addSchedule.jpg" file will be restored again. You already know from the above example that the path of the file is: ChronoSync_Help_v4.0/images.
  2. To restore this file without the entire path (ChronoSync_Help_v4.0/images) intact, choose "Restore" from the "Actions" menu, select "Specific Folder" on the "Restore from Archive" sheet and choose "Restore".
  3. You should now see the file browse sheet. Navigate to the location you wish to restore the selected file. Again, "addSchedule.jpg" will be restored to the Desktop. Make sure "Recreate parent folders" is NOT checked.
  1. Once you choose Select, a restore verification sheet will be displayed.
  2. The Finder window graphic below shows the file without its parent folders after restoring it. It restores just itself.
