Archive Handling

ChronoSync can maintain an 'archive', which is a repository of old files that were deleted or replaced during normal synchronization operations. Maintaining an archive is a good way to ensure that files, which are accidentally deleted or replaced, can be safely recovered. You can define how ChronoSync handles the archiving of files by using the "Archive Handling" option group.

Archive Handling options explained:

Limit duplicates in archive... - Older file copies will be removed after the specified, maximum limit is reached.

Purge files archived... - Older file copies that have resided in the archive longer than the specified time period will be deleted.

Keep at least... - Ensures a minimum number of files are kept in the archive regardless of how old they are.

Compress files added to archive - Files added to the archive will receive ZIP compression to reduce space. This has the potential to slow down synchronization performance, depending on the size of the files being compressed.

Copy files to archive instead of moving them - Files are copied to the archive instead of being moved. If you receive strange error messages when files are being copied to the archive, this option should be turned ON. Compressing files and copying them to the archive are mutually exclusive since a compress operation is essentially a copy operation, except the data is compressed as it is copied. If you are forced to copy instead of move due to filesystem limitations, you may as well consider the compress option.

Note: Copying files to archive may be time-consuming and should only be done if your destination filesystem does not properly support file-move operations.
