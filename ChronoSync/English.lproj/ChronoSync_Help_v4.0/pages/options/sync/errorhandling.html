Options

Reporting & Error Handling

You can define how ChronoSync reports information and/or deals with errors in the "Reporting & Error Handling" option group. Below, the Reporting & Error Handling options are explained:

Errors

Choose how errors should be handled:

Report to User - A sheet will display each time an error is encountered. This sheet requires user interaction. The synchronization will not continue until a selection is made and the sheet is dismissed.

Skip - Synchronization will resume as normal and nothing will be done with the offending files. If you will not be waiting around, or able to attend to each error that could possibly occur during a synchronization, this is the option you should choose. Once the synchronization is complete, you can browse the log to see the posted errors.

Exclude - Sometimes an error will repeatedly occur that simply cannot be corrected. For example, a file name may be too long to be synchronized with a target whose file system doesn’t support long file names. In this case, you may quickly and easily "Exclude" the offending file from future synchronizations, if it is deemed unimportant.

Abort - You can abort the entire process by choosing "Abort". This gives you a chance to immediately resolve the problem and then re-invoke the synchronization.

Roll-backs

Date roll-backs occur when the modification date of a file has gone back in time from its previous known setting. Under normal circumstances, this should never happen. It can accidentally happen, though, when you inadvertently replace a new file with an older one or it can also happen if something has gone wrong with clock synchronization between two computers or the system clock setting on your computer. This option will determine how roll-backs should be handled.

Note: This menu is not available if a bidirectional operation is chosen.

Ask User - A sheet will display each time a roll-back is encountered. This sheet requires user interaction. The synchronization will not continue until a selection is made and the sheet is dismissed.

Skip - Synchronization will resume as normal and nothing will be done with the offending files. Once the synchronization is complete, you can take care of the roll-back error. If you will not be waiting around, or able to attend to each error that could possibly occur during a synchronization, this is the option you should choose.

Exclude - Sometimes a roll-back error will repeatedly occur that simply cannot be corrected. In this case, you may quickly and easily "Exclude" the offending files from future synchronizations, if it is deemed unimportant.

Copy - The file of the source target is automatically copied to the destination target.

Abort - You can abort the entire process by choosing "Abort". This gives you a chance to immediately resolve the problem and then re-invoke the synchronization.

Conflicts

A conflict arises when two files have been determined to have both changed since the time of last synchronization. While one file will likely be more recent than the other, ChronoSync decides to err on the side of caution and flag these files as a conflict. This option will determine how conflicts should be handled.

Note: This menu is only available if a bidirectional operation is chosen.

Ask User - A sheet will display each time a conflict is encountered. This sheet requires user interaction. The synchronization will not continue until a selection is made and the sheet is dismissed.

Skip - Synchronization will resume as normal and nothing will be done with the offending files. Once the synchronization is complete, you can take care of the files in conflict. If you will not be waiting around, or able to attend to each error that could possibly occur during a synchronization, this is the option you should choose.

Exclude - Sometimes a conflict will repeatedly occur that simply cannot be corrected. In this case, you may quickly and easily "Exclude" the offending files from future synchronizations, if it is deemed unimportant.

Abort - You can abort the entire process by choosing "Abort". This gives you a chance to immediately resolve the problem and then re-invoke the synchronization.

Use Newer - If you wish to choose the newer of the two files for each conflict that arises, choose "Use Newer".

Use Older - If you wish to choose the older of the two files for each conflict that arises, choose "Use Older".

Use Left - If you wish to choose the left file of the two files that are in conflict, choose "Use Left".

Use Right - If you wish to choose the right file of the two files that are in conflict, choose "Use Right".

Automatic retry before handling errors - If an error occurs during synchronization, ChronoSync will retry twice (if necessary) before an error is reported. Some errors, such as a file in use by other applications, may get cleared up by retrying.

Display summary after synchronization completes - Displays a completion summary sheet after a synchronization.

Post more detailed information to the log - Posts extensive entries to the local log.

Ignore extended attribute and access control list errors - Errors generated when extended attributes and/or access control lists cannot be preserved will be ignored. Some filesystems and servers don't support extended attributes and/or access control lists and, therefore, these errors might be quite common. That is why you would want to ignore them.
