Sync Triggers

Sync Triggers indicate what file attributes ChronoSync monitors to detect that a file has changed. The most common attribute to monitor is the file's modification date - this date gets updated whenever a file's contents change. When ChronoSync detects a change in modification date, it will synchronize that file. This panel presents preset trigger modes, selected via a popup. There currently are four groups of settings: "Basic", "Basic w/ Attributes", "Extended" and "Custom".

  • Basic - Modification date changes will trigger a synchronization.
  • Basic w/ Attributes - Modification date and all simple metadata changes will trigger a synchronization.
  • Extended - Modification date, data size, all simple and most complex metadata changes will trigger a synchronization.
  • Custom - Can select your own options.

Sync Triggers options explained:

Modification date (with allowed time difference) - When two files are evaluated, the modification dates of those files are compared to determine if a change has occurred and which file has been changed more recently. The "allowed time difference" indicates the number of seconds difference that must exist between the two files before they are truly considered different.

File size - Changes in a file's size will synchronize.

Note: The "Modification date" and "File size" triggers will cause an entire file to get copied. All of the other triggers will result in just the appropriate metadata changes being copied.

Timestamps - Changes in the creation date of files or the creation and modification dates of folders will synchronize.

Finder flags - Attributes such as labels and the extension hiding flag are included in this grouping. There are other attributes but these are the most common. Any changes to these will synchronize.

HFS type/creator - Changes to HFS Type and Creator codes of a file will synchronize.

BSD flags - BSD Flags metadata changes of files and folders will synchronize. Examples are opaque, nodump, uchange.

Access permissions - Changes to the read/write privileges of a file will synchronize.

Locked status - A file can be either locked or unlocked. If locked, it is protected from changes. Changes to this state will cause a file to be synchronized. This setting is somewhat redundant with BSD flags, but not in all cases.

Owner/group - User/group ownership of the file will synchronize (requires admin access).

Note: The following "Complex Attributes" have a performance overhead. Setting them can slow performance.

Filename case - The upper/lower case changes of the filename will synchronize.

Extended Attributes - Changes to extended attributes will trigger an attribute synchronization. Extended attributes extend the basic attributes associated with files and directories in the file system. They are stored as name:data pairs associated with file system objects.

Access Control List - Changes to a file's access control list will trigger an attribute synchronization. Access control lists, or ACLs, are a way to control file permissions: who can do what to which files.

Comments - Changes to Finder/Spotlight comments will trigger an attribute synchronization. Under Leopard, this setting may be redundant with Extended Attributes since comments are stored as extended attributes. Exceptions would be if synchronizing with a non-Leopard machine or synchronizing files that had comments created on a pre-Leopard system (such as prior to upgrading Tiger to Leopard).

Always Trigger - Forces a file to always be copied. Allows a Synchronizer document to be created whereby the custom rules are the sole determinator of whether or not a file gets copied.

For most general purpose synchronizations, the default settings are adequate. If you change any of the "Sync Triggers" you should have an understanding of how the corresponding attribute affects your files.
