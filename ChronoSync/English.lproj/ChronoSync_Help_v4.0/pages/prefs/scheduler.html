Scheduler Preferences

Several scheduling options can be found on the Scheduler panel in "Preferences". Setting these options define the default scheduler behavior applied to every Synchronizer document. These options are listed below.

Scheduling Behavior:

Use background scheduler - Enable this option to run scheduled documents even if ChronoSync is not running.

Wake system to synchronize - Enable this option to automagically wake your Mac from sleep when a synchronization is scheduled to occur.

Run scheduled items sequentially - Enabling this option will allow only one scheduled synchronization to run at a time. When disabled, the scheduled synchronizations will run concurrently. This only applies when two or more synchronizations are scheduled to run at the same time. Sequentially is safer, but concurrently might offer some performance benefits.

Automatically hide Scheduler window - Determines whether or not the Scheduled Documents Manager window automatically hides itself whenever the application deactivates.

Missed Sync Behavior:

The background scheduler will detect when synchronizations were missed because the computer was off, the user was logged out, or the system was in sleep mode. The three options here tell the background scheduler how to deal with these missed synchronizations. It can either run them immediately, post an error to the scheduler log or, simply ignore them.

Notifications:

Send email to... - An email will be generated of the scheduled synchronization operation and sent to the email address(es) provided. Sending a notification can give you peace of mind that your synchronization ran. It can also point out errors so that you may address them.

Note: To send emails to multiple addresses, separate the addresses with commas.

Only send when errors have occurred - Will only generate an email if errors were encountered during the synchronization.

Send Test - Sends a simple email message to the specified address(es) to ensure that they are entered properly and that email has been setup correctly.

Post messages via Notification Center - Will post synchronization summary details to the Notification Center.

Only post when errors have occurred - Will only post to the Notification Center if an error was encountered during the scheduled synchronization.

Push notification to ChronoMonitor with ID - This will send a push notification to the ChronoMonitor app installed on your iPhone. Only post when errors have occurred - Will only generate a push notification if errors were encountered during the scheduled synchronization.

Send Test - Sends a simple notification to the specified device(s) to ensure that they are entered properly and that notification has been setup correctly.

ChronoSync defaults to sending email via Apple Mail. If you want to change this, go to the General panel in "Preferences". Aside from Apple Mail, there are other popular clients that are supported. ChronoSync can also send email directly without using an email client at all.
