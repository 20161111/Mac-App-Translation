Container Documents

A Container is a document that contains multiple ChronoSync documents. They allow you to control several ChronoSync documents as if they were one document.

Running a synchronization on a container will execute each contained document in order. You cannot run Trial Synchronizations with Container documents. If you wish to run a trial synchronization, it must be done on each individual Synchronizer document since one synchronization might affect the outcome of another.

Container documents have their own independent log and may be individually scheduled just like regular ChronoSync documents.

Documents may be added to or removed from the container by simply dragging them in and out or by selecting “Add...” or “Remove” from the toolbar. Just as with any other document based application, Container documents may be saved to disk.

A new "Untitled" Container document can be created by either:

  • Choosing "New" from the "File" menu and then selecting “Container”.
  • Choosing "Create a new container document" in the Document Organizer window.
  • Using the shortcut keys, SHIFT-COMMAND-N.

If you have the "Maintain documents folder:" preference setting enabled, when you open a new, untitled container document, a sheet drops down so that you can specify a filename, if you desire. The folder that is chosen is where the new document will be kept.
