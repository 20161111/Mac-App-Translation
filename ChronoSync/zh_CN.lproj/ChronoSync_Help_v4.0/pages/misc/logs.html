Miscellaneous

Logs

ChronoSync has the ability to maintain two separate log files - document and scheduler logs.

Document Logs

To view a document’s log, simply click the “Log” icon in the toolbar. The log for ChronoSync documents provides basic details about the document whenever it performs a synchronization. This includes summary information such as how many files were scanned, how many files were processed (copied) and how many errors may have occurred. The maximum log size is 4 MB. Document logs are available for both Synchronizer and Container documents.

Scheduler Logs

To view the scheduler log, simply click the “Log” icon at the upper left of the "Scheduled Documents Manager" window. This log contains a list of every attempt to to execute a scheduled document. The date and time of execution and the name of the document are listed. If any errors occurred, that fact is also reported. If a specific document is generating errors, you must open that document and inspect its own private log to determine what is going wrong.

Each log has the option of mirroring to an external text file. If you have a document log open and choose "Options", a sheet will drop down for you to choose a file to mirror the document's log to. Navigate to any location to keep a mirrored copy of your log file. You may also choose to set a maximum log file size.

You can also view "Errors Only" or "Notices Only" by filtering the entries using the "Filter" menu. When your synchronization is large and it has errors, this is a simple way to view errors quickly, without having to scroll through the other log content.

Note: Log files can grow quickly. Be aware of the size and location of this external file.

External logs may be viewed by the Console application.
