Reporting & Error Handling for Containers

You can define how Containers perform reporting and error handling by controlling options in the "Reporting & Error Handling" options panel.

Reporting & Error Handling options explained:

Disable user interaction for errors & warnings - Disables reporting of errors and warnings in the progress sheet window. Overrides the settings of individual Synchronizer documents that are contained. Errors are still posted to the log.

Continue after synchronization errors - Indicates that synchronization should continue with the next document if a contained synchronizer returns an error. If unchecked, the synchronization will stop with the Synchronizer document that generated the error.

Display summary after synchronization completes - Displays a completion summary sheet after a synchronization. Requires user interaction to dismiss.
