Options

Special File/Folder Handling

You can define how ChronoSync handles special types of files and/or situations using the "Special File/Folder Handling" option group. This panel presents preset handling modes, selected via a popup. There currently are three groups of settings. With "Basic" and "Basic w/ Verification" the checkbox controls are disabled. If you choose "Custom:", all the checkbox controls are enabled.

  • Basic - Standard file copy, no extra handling.
  • Basic w/ Verification - Standard file copy, data is verified, no extra handling.
  • Custom - Can select your own options.

Special File/Folder Handling options explained:

SmartScan (when available) - SmartScan can greatly increase performance when a sync or backup is configured to compare a very large number of files. It is only available for locally attached, fixed storage devices. It is not available for removable media or file servers. There are also a some rules for which it is not available. It works across ChronoAgent connections as long as the target volume is a fixed storage device attached to the ChronoAgent Mac. More details on SmartScan and when it should and should not be used can be found here.

NOTE: There are a handful of rules that require metadata that is not stored in the ChronoSync Difference Engine database and thus an OS API needs to be called for each and every file that is being tested. Doing this completely negates the benefits of SmartScan and would, in fact, make sync times much slower.

Verify copied data - Compare all copied data to the data in the original file. This option is not ideal for initial synchronizations or bootable backups due to the potential for a large number of files being copied. Having this option turned on will cause the synchronization to run slower than normal.

Disable safe-copy - Bypasses the process of copying files to a temporary file and then deleting the file being replaced after the copy has been safely completed. This is a more dangerous behavior since you risk losing the backed up copy of a file if an error occurs. It is a useful option, however, in cases where the target volume's size is constrained and it cannot accommodate large, temporary files.

Don't create empty folders - Prevents creation of empty folder hierarchies.

Package handling - 'Standard' treats packaged files as a whole. 'Merge' attempts to merge changes of the packaged files. 'Dissect' treats packaged folders as regular folders and examines their content.

Ignore Invisibles - Invisible files and folders will not be scanned if this option is checked.

Resolve aliases - Refer to the original item when scanning and copying. Activating this feature will copy the original file and not the alias. "Resolve aliases" should not be checked for most synchronizations and backups and only used in special circumstances. Only at root level will only resolve alias files located at the root level of the targets.

Auto-exclude new items - New files and folders added to a target will be automatically excluded during the next sync. Only at root level will only exclude files located at the root level of the targets.

Preserve comments - Preserves Finder/Spotlight comments associated with files and folders. Note that performing a synchronization with Preserve comments turned on will cause the synchronization to run slower than normal.

Preserve access control lists - Controls optional ACL preservation on filesystems that support them. It may be possible to preserve ACLs between two systems, especially in agent-based synchronization situations. But, because of different user/group relationships on the two systems, the end result may not be what is expected or desired. This is why you might need to turn this setting OFF.

Preserve extended attributes - Controls extended attributes on filesystems that support them. Turning this setting OFF will restrict EA preservation in certain scenarios. If restricted, any target that is not connected via ChronoAgent AND is not HFS+ formatted will NOT allow the writing of extended attributes.

NOTE: Having "Ignore invisibles" disabled while "Preserve comments" is enabled may be redundant. This is because the invisible '.DSStore' file, which contains the Finder/Spotlight Comments for the files in each folder, will likely get copied.
