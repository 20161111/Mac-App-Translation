Connection Profiles

Connections -- Advanced

The Advanced panel contains connection and security options. Connection options allow you to choose data compression or the ability to reconnect if dropped. Security options allow you to better protect your data when synchronizing to an agent. The wizard at the bottom of the window provides an explanation of how security affects your synchronization. Note that security isn't usually necessary if communicating over a closed, private network.

Connection options:

Data compression - Uses data compression, if supported. This setting is available for connections to agents and sharespaces.

Accessed on WAN - Indicates the connection is accessed over a wide area network (WAN) or a relatively low speed network.

Attempt reconnect on drops - When enabled and if supported by the agent, ChronoSync will attempt to reconnect to the agent if the connection was dropped or lost.

Optimal host routing - When enabled, ChronoSync will choose the most optimal route for data transfer when multiple routes to the remote host exist.

Limit throughput to... - Limits the network throughput of the connection to the specified value.

Custom timeouts:

Connection - The number of seconds to wait for a response after a connection request has been initiated.

Name resolution - The number of seconds to try and resolve a service name to a network address.

Std. Command - The number of seconds to wait for a response after a standard command has been sent to the remote service.

Async. Command - The number of seconds to wait for a response after an asynchronous command has been sent to the remote service.

Security:

Force all data to be encrypted - Determines if communication with the remote agent should be encrypted for enhanced security. Turning this ON will likely slow the synchronization.

Backwards compatible with older versions of ChronoAgent - When enabled, allows connections to older versions of ChronoAgent that may conform to weaker security standards. When OFF, ChronoSync will require remote ChronoAgents to employ at least the same level of security that is utilized by the current version of ChronoSync. Leaving this OFF is the more secure option but it may require remote ChronoAgents to be updated at the same time as ChronoSync, if a newer version of ChronoSync employs more advanced security techniques.
