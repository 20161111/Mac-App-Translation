Rules

Rules Overview

ChronoSync offers three different modes for the rule-base: "Simple", "Intermediate", and "Advanced". These modes are specified using the “Rule-base Mode“ popup menu at the top of the "Rules" panel.

It is important to understand that rules only determine what ChronoSync will consider for synchronization. This means that once a file passes a set of rules, ChronoSync will then examine the file to determine if it should be synchronized. If ChronoSync determines that the file in question has not been modified, it will not be synchronized. For example, it is not possible to create a rule that will always copy files with a “.jpg” extension regardless of whether or not they have changed (the exception to this is if you have the "Always Trigger" "Sync Trigger" turned ON).

Simple - The "Simple" rule-base mode has five predefined rules in it: “Displayed Name”, “Filename Extension”, “File Size”, “Date/Time Last Modified” and “OS Type/Creator Codes”. All five rules are disabled by default. You can enable just the desired rules and supply the necessary criteria. The "Simple" rule-base mode is the quickest and simplest way to introduce rule logic into your synchronizations.

Intermediate - The "Intermediate" rule-base mode has no pre-defined rules. Instead, you must explicitly add rules to the rule-base. Unlike "Simple" mode, you have more than five rules to choose from. Furthermore, you can add the same type of rule multiple times (with different criteria, of course). Thus you can add a rule that includes files with a “jpg” extension and another rule which includes files with a “tiff” extension. The "Intermediate" rule-base mode uses implied inclusion/exclusion logic. This means that files that pass the rules in the "Intermediate" rule-base mode are considered for synchronization while files that fail the rules are never considered for synchronization.

Advanced - The "Advanced" rule-base mode is similar to the "Intermediate" mode in that you can add any number or variety of rules. It differs from "Intermediate", however, in that it uses explicit inclusion/exclusion logic. This means that you can independently define rules that determine if a file should be considered for synchronization and/or excluded from or not considered for synchronization. Otherwise, "Advanced" mode is very similar to "Intermediate" mode.

Note: If a folder does not pass a rule, ChronoSync will not look inside that folder at other files to see if they pass the rule. ChronoSync will skip the folder and all of its contents.
