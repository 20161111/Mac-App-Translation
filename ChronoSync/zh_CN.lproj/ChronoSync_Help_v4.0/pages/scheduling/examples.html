Scheduling

Scheduling Examples

Scheduled synchronizations can be defined to run at very specific intervals at all hours of the day. With ChronoSync, it is simple to set up your schedule to run at any time that you wish. Here are some examples of how you can set up different intervals for scheduling synchronizations.

The SHIFT and COMMAND (Apple) keys can be of great use when setting up your schedules. If you hold down the SHIFT key, you are able to multi-select an adjacent range of values. If you hold down the COMMAND key, you are able to multi-select nonadjacent values.

EXAMPLE ONE:
Weekly, on Friday, every other hour at :00 and :30 past:

  1. Select "Weekly" from the "Run Sync." popup menu.
  2. Select "Friday".
  3. Hold down the COMMAND key. With your mouse, click on "12, 2, 4, 6..." and so on in the "Hours" section.
  4. Hold down the COMMAND key. With your mouse, click on "0" and "30" in the "Minutes" section.

EXAMPLE TWO:
Weekly, on Thursday and Friday, at 6:00, 6:01, 6:02, 6:03, 6:04 and 6:05 p.m.:

  1. Select "Weekly" from the "Run Sync." popup menu.
  2. Hold down the COMMAND key. Select "Thursday" and "Friday".
  3. With your mouse, click on "6 PM" in the "Hours" section.
  4. Hold down the SHIFT key. With your mouse, click on "0" in the "Minutes" section. Now, click on "5". Everything in between should become highlighted.

EXAMPLE THREE:
The 7th, 14th, 21st and 28th of every month at 8:00, 8:01, 8:02, 8:50, 8:51 and 8:52 a.m.:

  1. Select "Monthly" from the "Run Sync." popup menu.
  2. Hold down the COMMAND key. Select "7", "14", "21" and "28" in the "Days" section.
  3. With your mouse, click on "8 AM"in the "Hours" section.
  4. Hold down the SHIFT key. With your mouse, click on "0" in the "Minutes" section. Now, click on "2". Everything in between should become highlighted. Now hold down the COMMAND key. With your mouse, click on "50", "51", and "52".

Note: To de-select a selection, click on the selection while you have the COMMAND key held down.

Scheduling Options:

Only run if target volumes/agents are available - This item will be scheduled to run at the designated time but if the target volumes or agents are not presently available, it will not run. If this option is left OFF and the targets aren’t available, an error will be generated.

Prompt user before running - With this option ON, when a scheduled sync comes due, the user is prompted via the Notification Center as to whether or not the sync should be run. The prompt will remain visible until the user answers it, causing all subsequent syncs for the same item to be ignored until answered.

Run no more than once every... - When a document is scheduled to be run, ChronoSync will check the timestamp on the previously run synchronization. If the set value falls within the elapsed time, the synchronization will not run again.

Abort if still running at... - When you turn this ON, a time of day must be specified. If the scheduled sync is still running when that time of day is reached, it is suspended. This is intended to halt any long running syncs so they don’t interfere with daily activity, such as an overnight sync that may occasionally take a very long time to complete.

Retry on errors - If any errors occur during the synchronization, ChronoSync will reschedule the sync to run. Any scheduled item that shows "w/error" next to its last run time would qualify. "Retry on errors" takes precedence over "Run no more than once every...". This is new behavior for version 4.5.

Interval between retries - The number of minutes to wait before running the scheduled item again after an error. This time delay begins once the synchronization finishes.

Maximum retry attempts - The maximum number of retry attempts before giving up.
