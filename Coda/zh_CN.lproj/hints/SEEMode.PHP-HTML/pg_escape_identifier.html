pg_escape_identifier

Example

 string pg_escape_identifier ([ resource $connection ], string $data )

Description

pg_escape_identifier() escapes a identifier (e.g. table, field names) for quering the database. It returns an escaped identifier string for PostgreSQL server. pg_escape_identifier() adds double quotes before and after data. Users should not add double quotes. Use of this function is recommended for identifier parameters in query. For SQL literals (i.e. parameters except bytea), pg_escape_literal() or pg_escape_string() muse be used. For bytea type fields, pg_escape_bytea() must be used instead. Note: This function has internal escape code and can also be used with PostgreSQL 8.4 or less.

Return Values

A string containing the escaped data.
