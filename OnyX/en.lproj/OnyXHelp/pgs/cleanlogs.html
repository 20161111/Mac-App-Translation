Logs and Diagnostic Reports

When an application is open, it sends messages between the application, the system, and its components. This information is stored in log files. The system also creates logs and archives them during system maintenance operations. When an application does not respond, the details are saved in a diagnostic report that you can use to diagnose the issue or that you can send to the developer (if asked to do so).

You can use the Automation pane to remove all logs and diagnostic reports except the OnyX and bash Logs.

You can use the Logs pane of the Cleaning toolbar item to individually delete the logs and diagnostic reports.

■

Log files: This option deletes the applications Logs, Terminal Command Log, and some System Logs.

■

bash Log: You can delete the log that contains the list of commands entered in Terminal using this option.

■

System archived logs: When the periodic scripts (daily, weekly, and monthly) are run, log files are archived and compressed. There can be up to seven archives per log. You can remove all of the compressed archives and some system–saved logs using this option.

■

Instant message logs: This option removes the saved conversations of the most popular Instant Messaging applications. These applications are supported: Adium, AIM, Colloquy, Messages, Psi, Skype, Trillian, VChat, X-Chat Azure, and Yahoo! Messenger.

■

User Diagnostic Reports: You can use this option to delete all User Diagnostic Reports. The daily maintenance script also deletes these files but only those that have not been modified in the last 60 days.

■

System Diagnostic Reports: This option deletes any Diagnostic Reports created by the system and relaunches the syslogd daemon (a process that records the activities of the system).

■

Mobile Devices CrashReporter: This option deletes any Diagnostic Reports that have been created on your mobile device.
