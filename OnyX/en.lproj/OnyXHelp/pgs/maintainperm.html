Repair permissions

OS X incorporates a subsystem based on a UNIX–style operating system. In this kind of system, every file and folder on your hard disk has an associated set of permissions that determines who can read, write to, or execute it.

Many things that you install in OS X are installed from package files (.pkg ). Each time something is installed from a package, a file (.bom) is stored in a folder of the system or in a receipt file (.pkg). Each of those .bom files contains a list of the files installed by that package and the correct permissions for every file. When you use OnyX to verify and repair disk permissions, it reviews every .bom file and compares its list to the actual permissions of each listed file. If the permissions differ, OnyX corrects the difference.

Note: If you turn off SIP (System Integrity Protection) and if you remove receipts or .bom files from /System/Library/Receipts/, the files installed by the packages can neither be verified nor repaired. This is why you should not delete things from /System/Library/Receipts/.

Repairing permissions is the first thing to do as soon as you notice any issues with apps opening, have file access issues, or if your computer seems to be running unusually slow.

To repair the permissions:

Open OnyX. It will ask for your administrator password.

Select the Maintenance toolbar button, and click the Permissions tab.

Click the Execute button.

After the repair is completed, the result will appear in the text field present for this purpose.

You can use the Automation pane to repair permissions, but the result will not appear.
